# Лабораторная работа: настройка статической маршрутитизации и VPN между двумя локальными сетями

## Цель работы
Построить сеть с двумя локальными сегментами:
- левая сеть: `192.168.3.0/24`
- правая сеть: `192.168.4.0/24`

Необходимо:
1. Настроить IP-адресацию.
2. Прописать статические маршруты по умолчанию.
3. Настроить VPN между локальными сетями.
4. Проверить работоспособность сети.

---

## Состав сети

В схеме используются:
- 4 ПК: **PC0, PC1, PC2, PC3**
- 2 коммутатора: **Switch0, Switch1**
- 3 маршрутизатора: **Router1, Router0, Router2**

Коммутаторы нужны для подключения нескольких ПК к одному интерфейсу маршрутизатора:
- **Switch0** соединяет `PC0`, `PC1` и `Router1`
- **Switch1** соединяет `PC2`, `PC3` и `Router2`

---

## Схема сети

### Левая локальная сеть
- **PC0** — `192.168.3.2/24`, шлюз `192.168.3.1`
- **PC1** — `192.168.3.3/24`, шлюз `192.168.3.1`
- **Switch0** — коммутатор левой локальной сети
- **Router1 Fa0/0** — `192.168.3.1/24`

### Правая локальная сеть
- **PC2** — `192.168.4.2/24`, шлюз `192.168.4.1`
- **PC3** — `192.168.4.3/24`, шлюз `192.168.4.1`
- **Switch1** — коммутатор правой локальной сети
- **Router2 Fa0/0** — `192.168.4.1/24`

### Межроутерные соединения
- **Router1 Fa0/1** — `10.1.0.1/30`
- **Router0 Fa0/0** — `10.1.0.2/30`
- **Router2 Fa0/1** — `10.2.0.1/30`
- **Router0 Fa0/1** — `10.2.0.2/30`

## Подключение устройств

## Настройка коммутаторов

В этой работе коммутаторы используются как обычные устройства второго уровня. Так как в схеме по одной локальной сети с каждой стороны и VLAN не делятся, достаточно базовой настройки.

### Настройка Switch0

```cisco
enable
configure terminal
hostname Switch0
interface range fa0/1 - 3
 switchport mode access
 no shutdown
end
write memory
```

### Настройка Switch1

```cisco
enable
configure terminal
hostname Switch1
interface range fa0/1 - 3
 switchport mode access
 no shutdown
end
write memory
```

> Примечание: IP-адрес на коммутаторах в этой работе не обязателен, потому что они не участвуют в маршрутизации и VPN. Они только передают кадры внутри локальной сети.

---

## Настройка Router1

```cisco
enable
configure terminal
hostname Router1

interface fa0/0
 ip address 192.168.3.1 255.255.255.0
 no shutdown

interface fa0/1
 ip address 10.1.0.1 255.255.255.252
 no shutdown

ip route 0.0.0.0 0.0.0.0 10.1.0.2

access-list 110 permit ip 192.168.3.0 0.0.0.255 192.168.4.0 0.0.0.255

crypto isakmp policy 10
 encr aes
 hash sha
 authentication pre-share
 group 2
 lifetime 86400

crypto isakmp key cisco123 address 10.2.0.1

crypto ipsec transform-set MYSET esp-aes esp-sha-hmac

crypto map VPNMAP 10 ipsec-isakmp
 set peer 10.2.0.1
 set transform-set MYSET
 match address 110

interface fa0/1
 crypto map VPNMAP

end
write memory
```

## Проверка работы сети

### Проверка коммутаторов
Проверить, что порты на коммутаторах активны:

```cisco
show interfaces status
```

или

```cisco
show vlan brief
```

### Проверка базовой связности
На Router1:
```cisco
ping 10.1.0.2
```

На Router2:
```cisco
ping 10.2.0.2
```

На Router0:
```cisco
ping 10.1.0.1
ping 10.2.0.1
```

### Проверка VPN
С PC0 выполнить:
```bash
ping 192.168.4.2
```

Первый пакет может быть потерян, так как в этот момент устанавливается VPN-туннель. Последующие пакеты должны проходить успешно.

### Проверка состояния VPN
На Router1 и Router2:
```cisco
show crypto isakmp sa
show crypto ipsec sa
```

Ожидаемый результат:
- состояние `QM_IDLE`
- увеличение счетчиков `pkts encaps` и `pkts decaps`

## Объяснение выполненной работы

В данной лабораторной работе были настроены две локальные сети: `192.168.3.0/24` и `192.168.4.0/24`. Для подключения нескольких ПК с каждой стороны были использованы коммутаторы **Switch0** и **Switch1**. Они работают на втором уровне модели OSI и передают кадры внутри своей локальной сети.

Для связи между локальными сетями использовались три маршрутизатора. На крайних маршрутизаторах были заданы маршруты по умолчанию, направляющие трафик в сторону центрального маршрутизатора. На центральном маршрутизаторе были настроены статические маршруты в обе локальные сети.

Для защиты передаваемых данных между удалёнными локальными сетями был настроен VPN типа **site-to-site IPsec**. Были созданы:
- политика ISAKMP,
- общий предварительно согласованный ключ,
- набор преобразований IPsec,
- ACL для определения интересующего трафика,
- crypto map, привязанная к внешним интерфейсам маршрутизаторов.

В результате трафик между сетями `192.168.3.0/24` и `192.168.4.0/24` передаётся по защищённому туннелю, а коммутаторы обеспечивают подключение конечных устройств к маршрутизаторам.

## Вывод

В ходе работы была построена сеть с двумя удалёнными локальными сегментами, настроена IP-адресация, подключены коммутаторы доступа, реализована статическая маршрутизация по умолчанию и настроен защищённый VPN-канал IPsec между крайними маршрутизаторами. Работоспособность сети подтверждена успешной проверкой связности и просмотром состояния VPN-соединения.

## Примечание

Если в Packet Tracer оборудование не поддерживает алгоритмы `AES` и `SHA`, можно заменить настройки на более простые:

```cisco
crypto isakmp policy 10
 encr 3des
 hash md5
 authentication pre-share
 group 2
```

```cisco
crypto ipsec transform-set MYSET esp-3des esp-md5-hmac
```
